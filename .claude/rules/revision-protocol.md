# Revision Protocol

## 適用範圍

所有未通過品質門檻（< 8/10）的內容都必須遵守此退回修正協議。

---

## 退回觸發條件

### 強制退回（總分 < 7/10）

**article-analytics 必須退回**，不得放行。

### 建議修正（7-7.9/10）

**可發布，但建議修正**。由 article-consult 決定是否修正。

---

## 退回處理流程

### Step 1: Analytics 評估與標註

**article-analytics 必須提供**：

```markdown
## 品質評估：未達標 ({總分}/10)

### 評分明細
- 受眾吸引力：{X}/10
- 非受眾擴散力：{X}/10
- FB 適配度：{X}/10（若適用）
- Blog 適配度：{X}/10（若適用）

### 問題點（具體指出）
1. {問題類型}：{具體描述}
   - 問題位置：{段落/章節}
   - 影響：{為什麼這是問題}

2. {問題類型}：{具體描述}
   - 問題位置：{段落/章節}
   - 影響：{為什麼這是問題}

### 改善建議（可執行）
1. {建議 1}
   - 預期效果：提升 {X} 分數 {Y} 分

2. {建議 2}
   - 預期效果：提升 {X} 分數 {Y} 分

### 退回階段
Phase {X}：{具體 agent 名稱}

### 預期改善後分數
{預估分數}/10
```

---

### Step 2: 通知負責 Agent

**退回通知格式**：

```markdown
【退回通知】{文章主題}

退回原因：品質未達標 ({總分}/10)
退回到：Phase {X} - {agent 名稱}
退回時間：{YYYY-MM-DD HH:mm}

問題摘要：
- {問題 1}
- {問題 2}

改善建議：
- {建議 1}
- {建議 2}

期限：{X} 小時內完成修正
詳細評估報告：{連結或附件}
```

---

### Step 3: 負責 Agent 修正

**修正責任歸屬**：

| 問題類型 | 負責 Agent |
|----------|-----------|
| 主題定位不清、受眾錯誤 | article-planner |
| 研究資料不足、引用問題 | article-researcher |
| 內容空洞、缺乏深度、邏輯問題 | article-writer |
| 標題不吸睛、SEO 不佳 | copywriter |
| 文字不流暢、AI 味重 | article-editor |

**修正期限**：
- 輕微修正（7-7.9/10）：24 小時
- 重大修正（< 7/10）：48 小時

---

### Step 4: 重新送審

**修正完成後**：

```markdown
【修正完成通知】{文章主題}

修正版本：v{X}
修正時間：{YYYY-MM-DD HH:mm}
修正內容摘要：
- {修正 1}
- {修正 2}

請重新評估。
```

**article-analytics 重新評估**：
- ✅ 若達標（≥ 8/10）→ 通過
- ⚠️ 若 7-7.9/10 → 視情況決定
- ❌ 若仍 < 7/10 → 再次退回（記錄次數）

---

## 退回次數管理

### 退回次數限制

| 退回次數 | 處理方式 |
|---------|----------|
| **第 1 次** | 正常退回修正流程 |
| **第 2 次** | 升級處理：article-consult 介入檢視問題 |
| **第 3 次** | 重新企劃：回到 Phase 1 重新規劃 |

### 退回紀錄

每次退回必須記錄：

```markdown
# 退回紀錄：{文章主題}

| 次數 | 日期 | 分數 | 退回階段 | 主要問題 | 狀態 |
|------|------|------|---------|---------|------|
| 1 | 2026-02-13 | 6.5/10 | Phase 2 - Writer | 內容空洞 | 已修正 |
| 2 | 2026-02-14 | 7.2/10 | Phase 2 - Editor | AI 味重 | 修正中 |
```

---

## 特殊情況處理

### 用戶素材不足

**若退回原因是用戶素材不足**：

1. article-consult 通知用戶
2. 暫停專案，等待用戶補充素材
3. 收到素材後重新啟動

**不得強行撰寫**：嚴格遵守 `content-integrity.md`，不編造內容。

---

### 時效性內容

**若內容有時效性**：

- 可放寬至 7.5/10（但必須記錄原因）
- article-consult 批准後發布
- 事後補充修正版本

---

## 違反判定

- Analytics 發現 < 7/10 但未退回 → 違反
- 負責 agent 超過期限未修正 → 違反
- 未記錄退回次數 → 違反
- 第 3 次退回後仍繼續修正（應重新企劃）→ 違反
