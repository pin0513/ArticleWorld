# Handoff Protocol

## 適用範圍

所有 ArticleWorld 專案的階段交接都必須遵守此協議。

---

## Phase 1 → Phase 2 交接

### 交接時機

**觸發條件**（兩者必須滿足）：
1. article-planner 完成內容企劃
2. article-researcher 完成資料蒐集與引用整理

### 交付物檢查清單

**Planning 組必須交付**：

- [ ] **內容企劃文件**
  - 主題定義與核心論點
  - 目標受眾分析（年齡、職業、痛點）
  - 內容目標（告知、說服、娛樂）
  - 發布平台選擇（FB/IG/Blog/LinkedIn）

- [ ] **完整大綱**
  - 標題方向（3-5 個選項）
  - 文章結構（開場、主體、結尾）
  - 每段核心論點
  - 預計字數

- [ ] **研究資料包**
  - 引用來源清單（含連結）
  - 關鍵數據與統計
  - 專家觀點引用
  - 相關案例（公開可查證）

- [ ] **用戶素材清單**
  - 用戶提供的故事/經驗（若有）
  - 用戶提供的對話/訪談（若有）
  - 標註「可用」與「需補充」

### 交接確認

**Writing 組收到後必須確認**：
- ✅ 大綱是否清楚可執行
- ✅ 引用來源是否足夠
- ✅ 用戶素材是否充足（若不足，立即回報）

**若有問題**：立即回報 Planning 組，不得自行猜測補充。

---

## Phase 2 → Phase 3 交接

### 交接時機

**觸發條件**（三者必須滿足）：
1. article-writer 完成初稿
2. article-editor 完成審稿與去 AI 味檢測
3. copywriter 完成標題與行銷文案

### 交付物檢查清單

**Writing 組必須交付**：

- [ ] **初稿（article-writer）**
  - 完整文章內容
  - 符合大綱結構
  - 引用來源已標註
  - 字數符合規劃

- [ ] **編輯稿（article-editor）**
  - 潤飾後版本
  - 去 AI 味檢查報告
  - 標註修改處（track changes）
  - 邏輯與流暢度確認

- [ ] **行銷文案（copywriter）**
  - 最終標題（3 個選項）
  - Meta 描述（150-160 字元）
  - FB/IG 貼文文案
  - SEO 關鍵字清單

- [ ] **自我檢查報告**
  - 內容誠信確認（無編造內容）
  - 引用來源驗證
  - 目標受眾適配度自評

### 交接確認

**Quality 組收到後必須確認**：
- ✅ 所有交付物完整
- ✅ 格式符合規範
- ✅ 自我檢查報告已填寫

**若缺少任何交付物**：退回 Writing 組，不進入品質評估。

---

## 交接文件格式

每次交接必須產生交接文件：

```markdown
# 交接文件：{文章主題}

**交接日期**：{YYYY-MM-DD}
**專案編號**：{若有}
**交接階段**：Phase X → Phase Y

## 交付物清單
- [x] {交付物 1}
- [x] {交付物 2}
- [ ] {缺少的項目}

## 特殊說明
{若有特殊情況或注意事項}

## 確認簽收
- 交付方：{agent 名稱}
- 接收方：{agent 名稱}
- 確認時間：{時間}
```

---

## 違反判定

- 未完成交付物即交接 → 違反
- 缺少必要文件 → 違反
- 未產生交接文件 → 違反
- 接收方未確認即開始工作 → 違反
