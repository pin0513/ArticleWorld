# Quality Gate - 品質門檻規則

## 核心原則

> **所有內容必須達到 8/10 分以上才能發布。未達標則退回修正。**

---

## 評分維度

### 1. 受眾吸引力 (1-10)

**評估標準**：
- 標題是否吸睛？
- 開場是否抓住注意力？
- 內容是否與目標受眾相關？
- 語言是否符合受眾習慣？

**8分以上**：目標受眾看到會立即點開並讀完
**5-7分**：有吸引力但不夠強
**< 5分**：目標受眾不會停下來看

---

### 2. 非受眾擴散力 (1-10)

**評估標準**：
- 非目標受眾是否也能理解？
- 內容是否有分享價值？
- 是否有金句或亮點？
- 能否引發討論？

**8分以上**：非目標受眾也會分享轉發
**5-7分**：只有目標受眾會分享
**< 5分**：過於專業或小眾

---

### 3. 平台適配度

針對不同平台評分：

#### FB 適配度 (1-10)
- 標題長度：20-30字
- 內容結構：有分段、有小標
- 視覺元素：建議配圖
- 互動性：能否引發留言討論

#### IG 適配度 (1-10)
- 視覺優先：需要精美配圖
- 文案簡潔：前150字決定成敗
- Hashtag：10-15個相關標籤
- 限時動態：能否拆成多則貼文

#### Blog 適配度 (1-10)
- 深度：1500字以上
- SEO：關鍵字、Meta描述
- 可讀性：分段、小標、列表
- 價值：讀者願意收藏

#### LinkedIn 適配度 (1-10)
- 專業度：商業語言
- 洞察：有深度觀點
- 可信度：數據或案例支撐
- 價值：對職涯有幫助

---

## 總分計算

```
總分 = (受眾吸引力 + 非受眾擴散力 + 所有平台適配度) / 評分項目數
```

**範例**：
```
受眾吸引力：9/10
非受眾擴散力：8/10
FB 適配度：9/10
Blog 適配度：8/10
總分 = (9 + 8 + 9 + 8) / 4 = 8.5/10 ✅ 通過
```

---

## 品質門檻

| 總分 | 決定 | 行動 |
|------|------|------|
| **≥ 8/10** | ✅ 通過 | 進入發布流程 |
| **7-7.9/10** | ⚠️ 需改進 | 提供具體改善建議，輕微修正後可發布 |
| **< 7/10** | ❌ 退回 | 標註問題環節，重新執行對應 Phase |

---

## 退回處理流程

### 標註問題環節

| 問題類型 | 退回到 |
|----------|--------|
| 主題定位不清、受眾錯誤 | Phase 1: 企劃階段 |
| 內容空洞、缺乏深度 | Phase 2: 撰寫階段 |
| 文字不流暢、邏輯有誤 | Phase 2: 編輯階段 |
| 標題不夠吸睛、SEO不佳 | Phase 3: 行銷階段 |

### 改善建議格式

```markdown
## 品質評估結果：未達標 (6.5/10)

### 問題點
1. 受眾吸引力不足 (5/10)
   - 開場過於平淡，缺乏鉤子
   - 與目標受眾的痛點連結不夠強

2. FB 適配度偏低 (6/10)
   - 缺乏分段，一大段文字讀者會放棄
   - 沒有互動性問句

### 改善建議
1. 重寫開場：用反直覺陳述或故事鉤子
2. 加入3個小標，每段控制在150字內
3. 結尾加入開放式問句引發討論

### 退回階段
Phase 2: 撰寫階段（article-writer）

### 預期改善後分數
8/10
```

---

## 例外情況

以下情況可酌情放寬標準（但不低於7/10）：

1. **時效性內容**：快速回應熱門事件
2. **實驗性內容**：測試新風格或新主題
3. **內部文件**：團隊內部溝通用

**注意**：例外必須明確標註，並說明理由。

## 品質分析師職責

**article-analytics** 負責執行品質門檻檢查：

1. **客觀評分**：依據上述標準逐項評分
2. **具體建議**：不只說「不好」，要說「怎麼改」
3. **退回決策**：明確標註退回到哪個階段
4. **追蹤改善**：修正後重新評估

---

## 品質文化

> 「寧可多花一小時修改，也不要發出一篇6分的內容。」

品質不是檢查出來的，是做出來的：
- **企劃階段**：確保主題與受眾正確
- **撰寫階段**：確保內容有深度有人味
- **編輯階段**：確保邏輯清晰文字流暢
- **行銷階段**：確保標題與文案吸睛
- **分析階段**：確保達到發布標準

每個環節都把關，最終品質才有保證。
